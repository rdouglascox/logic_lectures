tree rules for gpli

tree rules for gpli

  • as in the transition from MPL to GPL we will build on our pre-existing tree rules
  • since == is part of the logical vocabulary, we need to say something about the tree rules for it

the closure rule

  • the first new rule is called the closure rule
  • this basically says that you can close a path if there is a proposition on it which asserts that something is not identical to itself!
  • the motivation for this rule should be obvious

new notation explained

  • recall that α(x_)\alpha(\underline{x}) stands for any arbitrary wff that has no free variables other than x_\underline{x} and that α(a_/x_)\alpha(\underline{a}/\underline{x}) stands for the wff that results from α(x_)\alpha(\underline{x}) by replacing all free occurrences of x_\underline{x} in α(x_)\alpha(\underline{x}) with the name a_\underline{a}
  • we use α(a_)\alpha(\underline{a}) to stand for any arbitrary wff in which the name a_\underline{a} occurs (one or more times)
  • we then use α(b_//a_)\alpha(\underline{b}//\underline{a}) to stand for any wff which results from α(a_)\alpha(\underline{a}) by replacing some occurrences of a_\underline{a} with the name b_\underline{b}

some examples using the new notation

  • if α(a)\alpha(a) is GaGa, then α(b//a)\alpha(b//a) is GbGb and α(c//a)\alpha(c//a) is GcGc
  • if α(a)\alpha(a) is RabRab, then α(b//a)\alpha(b//a) is RbbRbb and α(c//a)\alpha(c//a) is RcbRcb
  • if α(a)\alpha(a) is (Ga∨Rab)(Ga \lor Rab), then α(c//a)\alpha(c//a) is (Gc∨Rab)(Gc \lor Rab) or (Ga∨Rcb)(Ga \lor Rcb) or (Gc∨Rcb)(Gc \lor Rcb)
  • if α(a)\alpha(a) is (Ga→Rbb)(Ga \rightarrow Rbb), then α(a//b)\alpha(a//b) is (Ga→Rab)(Ga \rightarrow Rab) or (Ga→Rba)(Ga \rightarrow Rba) or (Ga→Raa)(Ga \rightarrow Raa)

finished trees, closed and saturated paths

finished trees, closed and saturated paths

  • a tree is finished when all paths are closed or saturated
  • a tree is saturated when no more rules can be applied to it
  • recall the qualification for the rule for the universal quantifier
  • we add the following qualification for saturation

saturation

  • saturation: a path is not saturated unless every application of SI that could be made on that path and that would result in the addition to the path of a formula that does not already appear on the path has been made

reading models off open paths

reading models off open paths

  • given a saturated open path, we can read off from it a model on which the proposition(s) at the top of the tree are true
  • the process involves three stages
    • stage one: construct a provisional domain and assignment of referents to names
    • stage two: trim the domain and assignments of referents to names to obtain a final domain and assignment of referents to names
    • stage three: assign extensions to predicates on the trimmed domain and assignments of referents
